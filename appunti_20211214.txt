- come funziona il web
- i principali tag html e accenno agli attributi

***
    client --> dispositivo che ha la possibilità di connettersi ad internet
    server --> computer costantemente collegato alla rete che "serve" i client
    internet --> tutto quello che c'è fra client e server e permette la loro comunicazione.

    su un server, un sito è una folder (root) nella quale sono presenti principalmete:
        - file .html --> CONTENUTO (il punto di ingresso è index.html)
        - file .css --> STILE (font, colori, margini, ecc)
        - file .js --> FUNZIONAMENTO (parte "magica")

    - sul mio browser, nella barra di ricerca, digito un indirizzo:
        client ---> http request ---> server
    
    la request contiene:
        - dominio: ad esempio www.facebook.it
        - action/method: GET

    Il server gestisce le varie request che gli arrivano:
        server ---> http response ---> client
    
    La response contiene:
        - pacchetto dei file .html, .css e .js della root del sito che abbiamo richiesto
        - response code: qualche esempio:
            - 200 "OK" --> la request è andata a buon fine
            - 404 "Not Found"
            - 500 "Internal Server Error"

    Il nostro browser (Chrome) ha al suo interno un "interprete" che:
        - legge in modo sequenziale il file;
        - lo rende graficamente comprensibile all'utente

    FRONTEND VS BACKEND
    I file che ci manda il server (html, css, js) fanno parte delle tecnologie FRONTEND:
        - vengono interpretate sulla nostra macchina dal browser --> sono interamente visibili e consultabili
        - considerato il punto precedente, è ovvio che NON devono contenere dati sensibili!

    Al contrario, i linguaggi BACKEND (C#, Java, PHP, ecc) vengono eseguiti sul server e non sono visibili dai client

    N.B: esercitarsi ad aprire pagine web ed ISPEZIONARLE:
        - shortcut CTRL+U = ispeziona pagina --> vediamo "in chiaro" il codice HTML della pagina 
        - shortcut F12 = strumenti sviluppatore --> scheda elements --> html e css
                                                --> scheda network --> files che il server ci invia nella response

***
    HTML = HyperText Markup Language = "Linguaggio di markup per ipertesti"

    "Linguaggio di markup.." = utilizza i markup (o marcatori, o etichette) o TAG
        TAG = parola chiave racchiusa da minore e maggiore anche dette parentesi angolari <>
    "..per ipertesti" = relativo a collegamenti ipertestuali:
        --> sono stati alla base della nascita di html e internet a fine 80/ inizio 90 in quanto permettono la navigazione fra risorse

    CREAZIONE DELLA STRUTTURA DEL SITO:
        - creare la folder root del sito
        - creare un file di testo e rinominarlo in index con estensione .html
        - scaricare Notepad++
        - aprire il file index con notepad

    STRUTTURA BASE DEI TAG:
        <nomeTag>Contenuto</nomeTag>
        dove:
            <nomeTag> = TAG DI APERTURA
            Contenuto = contenuto del tag; può anche essere un altro TAG (struttura a scatole cinesi)
            </nomeTag> = TAG DI CHIUSURA

    N.B. la struttura della pagina html è un annidamento di TAG (uno dentro l'altro come scatole cinesi)
        Quando osserviamo una pagina web, alleniamoci a suddividerla in "elementi che contengono altri elementi"

    STRUTTURA BASE DELLA PAGINA HTML:
        <!DOCTYPE html>
        <html>
            <head>
                <title>Questo è il testo visibile nel tab della scheda del browser</title>
            </head>
            <body>
                Qui andrà messo tutto il contenuto "visibile" della pagina web
            </body>
        </html>

    N.B.
        - indentare bene il codice! --> tasto TAB
        - Quando useremo altri IDE (VS, VS Code, ecc) indenteremo con delle shortcut, per ora facciamo a manella
        - In Notepad++ scrivete da subito sia il tag di apertura che quello di chiusura per evitare di dimenticarvelo dopo
        - l'indentazione serve a noi per non perderci all'interno del codice:
        - al browser non interessa se annidiamo il testo o no possiamo anche "minificare" (compattare) il file --> cercare "minify html file" su google

    Oltre i tag relativi alla struttura base della pagina abbiamo visto:
    I PRINCIPALI TAG:
        da <h1></h1> ad <h6></h6> --> headers --> intestazioni di primo(fino a sesto) livello
        <p></p> --> paragraph
        <ol><li></li></ol> per ordered list e list item
        <ul><li></li></ul> per unordered list e list item
        <strong></strong> per testo importante --> formattato in grassetto (alternativa a tag b)    
        <em></em> per testo enfatizzato --> formattato in corsivo (alternativa a tag i)
        <!-- --> per commenti (anche multilinea)

    ALCUNI TAG NON HANNO IL TAG DI CHIUSURA:
        <br> per i break --> per andare "a capo"
        <hr> per horizontal rule

    GLI ATTRIBUTI DEI TAG:
        i tag possono avere attributi ovvero:
            nomeAttributo="valore"
        ESEMPI:
            lang="it" da inserire nel tag di apertura di <html> --> <html>
            id="mioId" --> da inserire in un elemento per dargli un id che useremo poi in CSS (in una pagina un solo id univoco)
            class="miaClasse" --> da inserire in qualsiasi elemento per dargli una classe che useremo poi in CSS
            style="color:red; background-color:yellow" --> attributo usato per dare uno stile (lo gestiremo poi con CSS)

            N.B: il valore dell'attributo style sono una serie di coppie chiave-valore separati da ;
